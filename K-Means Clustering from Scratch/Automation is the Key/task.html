Description

It is now time to automate the whole process of the appropriate k search. As we mentioned before, we would rely on a limited "error" decrease as a criterion for an appropriate k â€” when the "error" decrease percentage is less than p, consider that the appropriate k has been reached.

One thing we have to make our mind on is whether we would return the actual k value that gives us the error decrease less than p or k-1. For this project, we believe taking k-1 is a more accurate choice.

As always, there is a question of whether the appropriate k value is the one that gives us the error decrease less than p or the previous one. We would stick to the first one, so if the value of k is what we found, we would return k-1 as an appropriate k.

In this stage, automate the process of finding an appropriate k. Create a loop that would increase k for your custom KMeans class starting from 1 until an appropriate k is found. Your ultimate criterion is an error decrease percentage less than p:

\[\text{abs}[(\text{Error}_{k-1} - \text{Error}_{k}) /\text{Error}_{k-1} \]

Once reached, return k-1.

Objectives
  1. Automate finding an appropriate k. One of the ways to do it is to create a function named find_appropriate_k.
  2. Print the resulting appropriate k for your X data. Use p=0.2 for your function.
